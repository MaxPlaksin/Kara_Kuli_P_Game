# Схема выборов по game_concept

Схема построена по файлу `game_concept.md.docx`: узлы — сцены и выборы, переходы — по ссылкам (стрелкам).

## Документ с полным текстом (удобно делиться)

**Файл: `game_scheme_full.html`**

- В каждом узле — **целиком** текст сцены/выбора из концепции.
- В конце каждого блока — блок **«Перейти по выбору»** со ссылками на следующие узлы.
- Сверху — оглавление по всем узлам. Один HTML: открыть в браузере, сохранить или отправить — всё работает без интернета.

**Пересобрать после правок в docx:** `python3 build_scheme_html.py`

## Как смотреть схему-картинку

- **В браузере:** открой `game_scheme.html` — отрисуется блок-схема.
- **Онлайн:** скопируй содержимое `game_scheme.mmd` на [mermaid.live](https://mermaid.live) — можно менять код и сразу видеть результат, экспортировать в PNG/SVG.

## Как редактировать

Редактируй **game_scheme.mmd** в любом редакторе.

- **Добавить узел:** новая строка, например  
  `NEW["Текст выбора или сцены"]`
- **Связать стрелкой:**  
  `Узел_источник --> NEW`  
  Переход «по стрелке» = выбор игрока ведёт в этот узел.
- **Перенос строки в подписи:** внутри текста используй `<br/>`.
- **Стиль узла (опционально):** в конце файла добавь  
  `style NEW fill:#e8f5e9`

В документе много веток «Продолжение Выбора 1.2.2.1.1…» — их можно по одной переносить в схему по тому же принципу: новый узел и стрелка от предыдущего выбора.

## Формат Mermaid

- `flowchart TB` — сверху вниз (можно заменить на `LR` для слева направо).
- `A["Текст"] --> B["Текст"]` — стрелка от A к B.
- Подписи в кавычках, квадратные скобки для блока.
